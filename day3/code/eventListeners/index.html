Demo: Event Listeners

Reageren op gebruikersacties (clicks, hover, input, etc.)

Voor deze demo:

Alle voorbeelden werken out-of-the-box! Klik op knoppen en zie wat er gebeurt.

Open DevTools Console (F12) om de event logs te zien.

Demo 1: Button Click Event

JavaScript Code:
// Selecteer de button let button = document.getElementById('simple-button'); // Voeg event listener toe button.addEventListener('click', function() { alert('Button geklikt!'); console.log('Click event triggered!'); });
Wat gebeurt er?
  1. JavaScript "luistert" naar click events op de button
  2. Als gebruiker klikt, wordt de functie uitgevoerd
  3. De pagina refresht NIET!

Demo 2: Click Counter

Elke click update de counter - zonder page refresh!

0
JavaScript Code:
let count = 0; let counterDisplay = document.getElementById('counter'); document.getElementById('increment-btn').addEventListener('click', function() { count++; counterDisplay.textContent = count; console.log('Count:', count); }); document.getElementById('decrement-btn').addEventListener('click', function() { count--; counterDisplay.textContent = count; }); document.getElementById('reset-btn').addEventListener('click', function() { count = 0; counterDisplay.textContent = count; });
Key Points:
  • We slaan state op in een variabele (count)
  • Bij elke click updaten we de state
  • We updaten de DOM om de nieuwe state te tonen
  • Geen page refresh!

Demo 3: Input Events (Real-time)

Reageer op typing - real-time!

Hallo, ...
JavaScript Code:
let input = document.getElementById('name-input'); let output = document.getElementById('greeting-output'); input.addEventListener('input', function(event) { let name = event.target.value; output.textContent = 'Hallo, ' + (name || '...'); console.log('Input changed:', name); });
Event Object:

event.target geeft toegang tot het element dat de event triggerde

event.target.value is de waarde van de input

Demo 4: Mouse Events (Hover, Enter, Leave)

Hover over mij!
Wacht op mouse...
JavaScript Code:
let box = document.getElementById('hover-box'); let output = document.getElementById('mouse-output'); box.addEventListener('mouseenter', function() { box.style.background = '#2ecc71'; box.textContent = 'Je bent er!'; output.textContent = 'Mouse entered!'; }); box.addEventListener('mouseleave', function() { box.style.background = '#3498db'; box.textContent = 'Hover over mij!'; output.textContent = 'Mouse left!'; }); box.addEventListener('click', function() { box.style.background = '#e74c3c'; output.textContent = 'Clicked!'; });

Demo 5: Form Submit BELANGRIJK!

Let op: preventDefault()

Zonder preventDefault() doet de form een normale submit = page refresh!

Eerst: ZONDER preventDefault()

Deze form doet een normale submit (page refresht):

Dan: MET preventDefault()

Deze form gebruikt JavaScript (page refresht NIET):

JavaScript Code (de GOEDE manier):
let form = document.getElementById('good-form'); form.addEventListener('submit', function(event) { event.preventDefault(); // BELANGRIJK! Stop normale submit // Haal data op let name = document.getElementById('form-name').value; let email = document.getElementById('form-email').value; // Toon result (later: AJAX request hier!) let output = document.getElementById('form-output'); output.textContent = 'Form submitted!\nNaam: ' + name + '\nEmail: ' + email; output.style.display = 'block'; console.log('Form data:', {name, email}); });
Waarom preventDefault()?
  • Zonder: browser doet normale form submit → page refresh
  • Met: we kunnen JavaScript gebruiken → AJAX, validation, etc.
  • Dit is de basis voor Single-Page Applications!
